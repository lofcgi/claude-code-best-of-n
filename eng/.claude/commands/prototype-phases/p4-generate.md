# Phase 4: Section-by-Section Code Generation (Fundamental Change)

> **⛔ v0 removed. Claude Code generates code directly.**
> Generated by combining reference blueprints + prebuilt components + design tokens.

---

## Core Principle: Section-by-Section Generation

**Current Problem**: Generating entire page at once (1000 lines) → inline component redefinition, uneven visual density

**Change**:
1. Generate section components in the `components/sections/` directory
2. Generate each section as an individual file → page only imports them
3. **Inline component definitions strictly prohibited** — only imports from `components/premium/` or `components/sections/` are allowed

### ⛔ Absolute Prohibition Rules
```
No function definitions inside page files (only export default allowed)
```

All section components must be defined in separate files and imported into the page.

## App Page Generation Rules

- Import colors/images from `design-tokens.ts` (no inline hex)
- Import UI components from `components/premium/`
- Read reference screenshots from `prototypes/references/` to match visual quality
- App page premium requirements:
  - Sidebar/Nav: glass-morphism (backdrop-blur-md, bg-white/5)
  - Cards: must have hover effects — border glow or cursor-following spotlight
  - Empty states: CSS abstract shapes — plain text only is prohibited
  - Micro-interactions: button press scale(0.97), input focus primary glow ring
  - All animations: spring physics — no linear

## Landing Page Generation Rules

### Page File Structure (Example)
```tsx
// app/a/page.tsx — imports only, no function definitions
import { HeroBeams } from "@/components/sections/hero-beams";
import { TrustMetrics } from "@/components/sections/trust-metrics";
import { FeatureBento } from "@/components/sections/feature-bento";
import { ComparisonTable } from "@/components/sections/comparison-table";
// ...

import { palettes, images } from "@/lib/design-tokens";

export default function LandingA() {
  const p = palettes.a;
  const img = images.a;

  return (
    <main>
      <HeroBeams palette={p} images={img} />
      <TrustMetrics palette={p} />
      <FeatureBento palette={p} images={img} />
      <ComparisonTable palette={p} />
      {/* ... minimum 8 sections */}
    </main>
  );
}
```

### Copywriting (copy-guide.md reference required)
- **Headlines**: 2-4 words, brand-specific expressions (no cliches)
- **Subtext**: 1 sentence, 15-20 words, include specific numbers
- **Feature names**: 2-3 words, brand terminology (no generic nouns)
- **CTA**: verb + benefit ("Start Dubbing Now" not "Get Started")
- **Numbers**: be specific (10,847 not 10,000, 98.5% not 99%)

### Visual Architecture (Common to all sections, non-negotiable)
- Hero: 5-layer architecture (visual-architecture.md)
- Hero headline: text-6xl md:text-8xl font-black
- GradientText required
- CTA: glow box-shadow 30px+ 60px dual layer
- BrowserMockup: 3D perspective or tinted shadow
- All sections: use SectionWrapper (minimum 2-layer background)
- Section padding: py-40 minimum
- Animations: spring physics only
- Images: minimum 5 per page
- Avatars: minimum 3 (64px, Unsplash)
- Feature cards: images required (text+icon only is prohibited)

### Visual Asset Requirements
- **Minimum 5 images (Unsplash/screenshots) per page**
- **Minimum 1 BrowserMockup** (app preview)
- **Feature cards with only text+icons are prohibited**
- **Minimum 3 avatar images** (testimonial)

## Phase 4.5: App Screenshots → Landing Page Injection

> This phase runs after app page generation is complete and before landing page generation.

1. **Start dev server**:
   ```bash
   cd prototypes/_app && npm run dev &
   ```
   Wait 3 seconds for server startup

2. **Take screenshots of each app page with Playwright**:
   - Viewport 1280x800
   - `browser_navigate` → `browser_take_screenshot`
   - Save to: `public/screenshots/{concept}-app.png`

3. **Display the corresponding screenshot inside BrowserMockup in each landing's Hero/Feature**:
   ```tsx
   <BrowserMockup url="app.{projectname}.ai">
     <img src="/screenshots/a-app.png" alt="{Project Name} App" className="w-full" />
   </BrowserMockup>
   ```

4. Stop dev server
